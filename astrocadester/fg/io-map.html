Astrocade Arcade Field Guide · Chapter 02

I/O Map

The port-level control panel for Bally/Midway Astrocade arcade software: video registers, sound/I/O direction traps, cabinet inputs, Magic expansion, protected RAM enable, and the Wizard/Gorf pattern-board port block.

⟵ Machine Model ⌂ Video System ⟶

Why this chapter matters

The Astrocade arcade I/O map is compact, but it is easy to misread because the Z80 has a separate I/O address space, MAME models some ports with high-address-bit selection, and several numeric port values have different meanings depending on whether the CPU is reading or writing.

A good arcade include file must therefore avoid one flat list of names like PORT_10. The safer model is direction-aware naming: PORT_TONMO when you write sound register 0, but PORT_COIN_DOOR or PORT_INPUT10 when you read the same numeric port on WoW-style wiring.

The big idea The I/O map is not just “addresses.” It is a set of hardware conversations. OUT ($10),A talks to the sound chip. IN A,($10) asks the sound/I/O hardware and board muxes for input state. Same low number, different bus direction, different meaning.

MAME backbone

This chapter uses MAME as the backbone, then overlays the arcade-oriented HW-HVGLIB.H.h names so code and documentation stay in sync. For the Wizard of Wor / Gorf class with the mono pattern-board extension, the practical high-level map is: 

00-0F  video registers
10-1F  sound/I/O reads
10-18  sound writes
19     expand register
78-7E  pattern board writes
A55B   protected RAM enable
Range MAME model Field-guide meaning Primary include names
$00-$0F Video register read/write block. Palette, video mode, blanking/split, Magic mode, interrupt feedback/mode/scanline. PORT_COL0R..PORT_SCANLINE
$10-$1F Astrocade sound/I/O reads. Input reads through the custom I/O chip and arcade board muxes. PORT_INPUT10, PORT_PLAYER2, PORT_PLAYER1, PORT_INPUT13, PORT_POT0..3
$10-$18 Astrocade sound writes. Master oscillator, tone generators, vibrato, volumes/noise, sound block writes. PORT_TONMO..PORT_SNDBX
$19 Expand register write. Magic expander source value; affects Magic write expansion/bit manipulation. PORT_XPAND / XPAND
$78-$7E Mono pattern-board write block. External pattern-board command/address/register block used by WoW/Gorf-style hardware. PORT_PBLINADRL..PORT_PBXR
$A55B Protected-RAM enable write. Unlocks the next protected RAM access. On Z80 I/O, high address bits can participate in decode. PORT_PROTECT=$5B, PROTECT_KEY_UNLOCK=$A5
Port-address warning MAME shows $A55B for the protected-RAM enable because its I/O map can decode more than the low 8 bits. Z80 source often writes OUT ($5B),A with A=$A5. That is not a contradiction; it is a decode detail you should document every time it appears.

Direction matters: read names and write names are not interchangeable

The arcade include intentionally says that several port numbers have different meanings for IN versus OUT. This is especially important in the $10-$1F range, where writes are sound registers but reads return I/O-chip or muxed cabinet/input data.

Write-side meaning

ld   a,$80
out  (PORT_TONMO),a    ; $10 = sound master oscillator write

ld   a,$0F
out  (PORT_VOLAB),a    ; $16 = tone A/B volume write

When writing, $10-$18 are sound-control registers on the Astrocade custom I/O chip.

Read-side meaning

in   a,(PORT_INPUT10)  ; $10 = WoW coin/start/input-side read
in   a,(PORT_PLAYER1)  ; $12 = WoW player 1 input-side read

When reading, the arcade board routes input data through the sound/I/O chip and external muxing.

Best practice Use names that describe the intended bus direction. Do not write new code that says out ($10),a unless you are doing a tiny one-off experiment. In real source, prefer out (PORT_TONMO),a or in a,(PORT_INPUT10).

Video register block: $00-$0F

The low sixteen ports are the Astrocade video register window. In the arcade field guide, treat these as the first part of every startup sequence: color latches, video mode, palette split, height/blanking, Magic mode, and interrupt controls.

Color latches: $00-$07

HW-HVGLIB.H.h keeps the classic HVGLIB color names but makes the direction explicit with PORT_ names. The include describes eight hardware color latches: $00-$03 for right-side palette colors 0-3 and $04-$07 for left-side palette colors 0-3.

$01 PORT_COL1R COL1R Right-side color register 1.
$02 PORT_COL2R COL2R Right-side color register 2.
$03 PORT_COL3R COL3R Right-side color register 3.
$04 PORT_COL0L COL0L Left-side color register 0.
$05 PORT_COL1L COL1L Left-side color register 1.
$06 PORT_COL2L COL2L Left-side color register 2.
$07 PORT_COL3L COL3L Left-side color register 3.

Video, Magic, and interrupt control: $08-$0F

$08 PORT_VIDEO_MODE CONCM Commercial/video mode control. Also has read-side status alias PORT_INT_STATUS/INTST.
$09 PORT_PAL_SPLIT HORCB Horizontal color boundary / palette split control.

Startup implication A minimal arcade ROM should set palette latches, video mode, height/blanking, Magic mode, and interrupt state before enabling interrupts or drawing anything that depends on stable video behavior.

Sound writes: $10-$18

The Astrocade custom I/O chip is also the sound chip. MAME documents eight sound registers, and the arcade include maps those write-side names onto ports $10-$18. Register $18 is kept as a sound-block multi-port compatibility name, while the core eight chip registers are $10-$17.

Port Preferred name Legacy alias Sound register role
$10 PORT_TONMO TONMO Register 0: master oscillator frequency.
$11 PORT_TONEA TONEA Register 1: tone generator A frequency.
$12 PORT_TONEB TONEB Register 2: tone generator B frequency.
$13 PORT_TONEC TONEC Register 3: tone generator C frequency.
$14 PORT_VIBRA VIBRA Register 4: vibrato speed/depth.
$15 PORT_VOLC VOLC Register 5: noise AM/mux and tone C volume.
$16 PORT_VOLAB VOLAB Register 6: tone B volume in high nibble, tone A volume in low nibble.
$17 PORT_VOLN VOLN Register 7: noise volume.
$18 PORT_SNDBX SNDBX Sound block / multi-port compatibility symbol.
The $15 collision The include deliberately documents a collision: console HVGLIB names OUT $15 as VOLC, but WoW-style arcade code also uses PORT_LATCH=$15 for cabinet/latch behavior. That is not something to “fix” with one universal name. It is exactly why direction and board context belong in the symbol name.

Input reads: $10-$1F

The read side of the $10-$1F range is where console assumptions are most dangerous. MAME’s sound/I/O device comments explain that home Astrocade systems logically tie SO/SI to the same input matrix, while arcade games route SI lines through external multiplexers and use SO strobes for unrelated outputs. In plain terms: the arcade cabinet does not behave like four simple console hand controllers.

Port Preferred include name Generic alias WoW-style interpretation
$10 PORT_COIN_DOOR / PORT_INPUT10 SW0 Coin/start/service style input group on current WoW experiments.
$11 PORT_PLAYER2 SW1 Player 2 input-side group.
$12 PORT_PLAYER1 SW2 Player 1 input-side group.
$13 PORT_INPUT13 SW3 Additional input/DIP/status group; verify per board.
$1C-$1F PORT_POT0..PORT_POT3 POT0..POT3 Console-style pot aliases kept for source compatibility; verify availability on the arcade board.

Input bit conventions used by our include

The include keeps both HVGLIB-style control bit names and project-style aliases. It also documents the active-low convention for matrix-style inputs: 0 = pressed/closed, 1 = open. That matches the way our polling code generally sanitizes joystick state before handing it to game logic. 

BIT_UP       equ 0
BIT_DOWN     equ 1
BIT_LEFT     equ 2
BIT_RIGHT    equ 3
BIT_TRIGGER  equ 4
BIT_START1   equ 5    ; normally on port $10
BIT_START2   equ 5    ; normally on port $11
Debugger habit When an input seems stuck, trace reads from $10-$13 and look at both the raw port value and your sanitized version. A bug can be in MAME mapping, board assumptions, active-low inversion, bit masks, or simply reading the wrong player port.

Expand register: $19

Port $19 is the Magic expander register, named PORT_XPAND or XPAND. It works with the Magic register and Magic write window. Conceptually, it is part of the graphics data path rather than the ordinary video-control block.

When Magic expansion is enabled, writes into Magic memory can expand compact source data into wider pixel patterns. This is one of the places where the Astrocade hardware stops looking like a plain framebuffer and starts acting like a small, odd blitter. 

; Conceptual use: exact mode/data depends on the Magic chapter.
ld   a,%11110000
out  (PORT_XPAND),a

ld   a,MAGIC_EXPAND | MAGIC_OR
out  (PORT_MAGIC),a

; Later writes into the Magic window use the current expand/magic state.
Chapter boundary This page identifies $19 and how to name it. The deep rules for expand tables, shifts, rotate, OR/XOR, and Magic-memory writes belong in the Magic System chapter.

Pattern board: $78-$7E, plus project alias $7F

The Wizard/Gorf neighborhood adds a pattern-board port block. MAME’s mono pattern-board map writes $78-$7E to the pattern-board handler and uses $A55B for protected RAM enable. Our include also names $7F as PORT_PBOR/PBOR because current project notes and older experimental code treat the cleaner pattern-board block as $78-$7F.

Port Preferred name Short alias Field-guide role
$78 PORT_PBLINADRL PBLINADRL Pattern-board line/address low.
$79 PORT_PBLINADRH PBLINADRH Pattern-board line/address high.
$7A PORT_PBCTL PBCTL Pattern-board control.
$7B PORT_PBROT PBROT Pattern-board rotate / area low compatibility alias.
$7C PORT_PBMASK PBMASK Pattern-board mask / area high compatibility alias.
$7D PORT_PBMR PBMR Pattern-board Magic-register style control / width compatibility alias.
$7E PORT_PBXR PBXR Pattern-board X register / height compatibility alias.
$7F PORT_PBOR PBOR Project alias for OR/output behavior; verify against schematic/original routines before treating as MAME-backed.
Evidence tag Treat $78-$7E as MAME-backed for the mono pattern-board handler. Treat $7F/PBOR as project/schematic/ROM-observed territory until this guide pins it down with a specific schematic page or ROM trace.

Protected RAM enable: $A55B / PORT_PROTECT

Protected RAM is not enabled by writing data directly into protected RAM. First, software hits the protection enable path, then performs the protected RAM access. MAME models the enable write at $A55B for the mono pattern-board map. The include provides PORT_PROTECT=$5B and PROTECT_KEY_UNLOCK=$A5 because this is how the operation appears naturally in Z80 source. 

PORT_PROTECT       equ $5B
PROTECT_KEY_UNLOCK equ $A5

; Conceptual protected write:
ld   a,PROTECT_KEY_UNLOCK
out  (PORT_PROTECT),a
ld   a,new_value
ld   (RAM_PROTECTED_START),a

The important software habit is to keep protected writes tight and intentional. Do not leave the unlock separated from the write by a long call chain. Do not use protected RAM for ordinary object state. Do not hide this behind misleading names like SafeWrite unless the routine is tiny and obviously writes exactly one byte.

HW-HVGLIB naming recommendations

The arcade include is doing two jobs at once: preserving useful HVGLIB compatibility names and preventing console assumptions from leaking into arcade code. The safest convention is:

When writing new code Use this style Reason
Hardware ports PORT_* Makes it obvious the symbol is an I/O port, not RAM or ROM.
Legacy-compatible code CONCM, MAGIC, XPAND, TONMO Useful when adapting code derived from HVGLIB or console examples.
Inputs PORT_PLAYER1, PORT_PLAYER2, PORT_INPUT10 Avoids pretending WoW wiring is the same as console controller order.
Pattern board PORT_PB* or PB* Keeps external pattern hardware distinct from core Astrocade Magic.
Magic modes MAGIC_OR, MAGIC_XOR, MRXOR_MASK Self-documents bit meanings instead of scattering hex constants.

Code patterns

Clear direction-aware examples

; Read cabinet/input-side port $10.
; This is NOT a sound write.
in   a,(PORT_INPUT10)
ld   (COIN_INPUT),a

; Write sound register 0 at numeric port $10.
; This is NOT an input read.
ld   a,$80
out  (PORT_TONMO),a

Palette setup shape

; Example only: values depend on the chosen Astrocade color map.
ld   a,$00
out  (PORT_COL0R),a
out  (PORT_COL0L),a

ld   a,$63
out  (PORT_COL3R),a
out  (PORT_COL3L),a

Input polling shape

PollArcadeInputs:
    in   a,(PORT_PLAYER1)
    ld   (P1_INPUT),a

    in   a,(PORT_PLAYER2)
    ld   (P2_INPUT),a

    in   a,(PORT_INPUT10)
    ld   (COIN_INPUT),a
    ret

Pattern-board write shape

; Conceptual only. Exact sequencing belongs in the pattern-board chapter.
ld   a,low pattern_addr
out  (PORT_PBLINADRL),a

ld   a,high pattern_addr
out  (PORT_PBLINADRH),a

ld   a,control_bits
out  (PORT_PBCTL),a
