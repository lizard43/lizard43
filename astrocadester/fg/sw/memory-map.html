Memory Maps

Wizard of Wor·Gorf·Robby Roto·Space Zap

Z80 address space · Wizard of Wor
Building memory map…
Stack: set SP at $E000 so pushes grow down into high work RAM Protected RAM: only 64 bytes, reserve for credits, latches, and tiny bookkeeping. Magic: reads from $0000-$3FFF are ROM reads; writes are graphics operations.
Z80 address space · Gorf

Gorf has high NVRAM and work RAM.

Building memory map…
Stack: set SP at $E800 so pushes grow down into high work RAM NVRAM: High non-volatile/protected storage instead of the WoW-style $D000 block. Magic: reads from $0000-$3FFF are ROM reads; writes are graphics operations.
Z80 address space · Space Zap

Space Zap is a simple map with small work RAM and upper open-bus regions. It does not use upper ROM expansion or dynamic banking, so large parts of the upper address space remain unmapped.

Building memory map…
Stack: set SP at $E800 so pushes grow down into high work RAM Protected RAM: mapped at $D000-$D03F. Magic: reads from $0000-$3FFF are ROM reads; writes are graphics operations.
Z80 address space · Robby Roto

Robby Roto has the standard low ROM/Magic and framebuffer regions, uses the upper half of the Z80 address space for fixed game ROM at $8000-$DFFF, high protected/NVRAM storage at $E000-$E7FF, and ordinary work RAM/stack space at $E800-$FFFF

Building memory map…
Stack: set SP at $0000 so pushes grow down into high work RAM Protected RAM: protected window moved up to $E000-$E1FF NVRAM: Robby Roto keeps NVRAM/protected storage at $E000-$E7FF while $8000-$DFFF is used as 24 KB of upper ROM Magic: reads from $0000-$3FFF are ROM reads; writes are graphics operations.
