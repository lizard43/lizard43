notes

This is a collection of resources gathered from everywhere about writing Z80 software for the arcade (not the console) Astrocade platform. Hardware games the arcade ganes like:
Wizard of Wor, Gorf, Robby Roto, Space Zap, and their similar 5-6 card-set cousins.

While writing software for the arcade Astrocade has a lot of similarities to the console cartridge development, there are also a lot of differences and gotchas.

This guide is a practical notebook for ROM hacking, new development, emulator experiments, and real-hardware sanity checks on Bally/Midway Astrocade arcade hardware.

why is this hard?

Writing 8-bit Z80 is not hard, it's the figuring the underlying platform and the details that make it tough.

The Astrocade arcade platform was not a consumer development target. Bally/Midway and Dave Nutting Associates (DNA) built commercial embedded games, shipped ROMs, and documented the machines mainly so operators could keep them running.

The result is an archaeological dig.

Resources like BallyAlley have tons (and tons) of great info but most is targeted to the Astrocade console and not the Astrocade arcade hardware platform.

The Bally/Midway Service manuals are good but they are targeted for the technician to make repairs.

MAME repository is a fantastic resource both for perusing the repo source code and using the MAME debugger to watch how the emulator interprets the code.

Robby Roto, Wizard of Wor and Gorf disassemblies tell you how commercial games coded against the arcade hardware.

Not console Astrocade

The console system knowledge matters, especially Magic, palette, and sound concepts. But arcade machines have a different memory map, don't use interpertive mode, have different input routing, pattern boards, protected RAM behavior, speech hardware, and hi-res video.

sources

BallyAlley Your first stop. Master astrocade archive, scans, historical notes, console and arcade cross-links. But beware, not all aapplies to arcade hardware
8bitworkshop Learn how classic game hardware worked. Write code and see it run instantly. Use this to understand concepts. Be careful, console code will not run on arcade hardware and visa-versa.
MAME Uncover memory maps, I/O maps, input definitions, emulated behavior. Sound register behavior and arcade SI/SO.
Service manuals Schematics, wiring, card racks, DIP switches, edge connectors, board layouts.
Robby Roto paper and disassembly The paper is a good overview of the arcade system while the github repo contains the disassembled asm source
Console Astrocade docs Magic, palette, sound, Z80 conventions, original chip concepts.

The arcade Astrocade family

The same design lineage appears across multiple Bally/Midway titles, with variations in ROM layout, RAM, input routing, speech, and pattern-board usage.

Wizard of Wor (1980) Gorf (1981) Robby Roto (1981) Seawolf II (1978) Extra Bases (1980) Space Zap (1980) Demons and Dragons (1982) Professor Pac-Man (1983) 10 Pin Deluxe (1984) 10 Pin Champ / Strikes and Spares (1985)

This guide focuses on Wizard of Wor Gorf Robby Roto
