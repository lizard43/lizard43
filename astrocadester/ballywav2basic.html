BALLY ON-BOARD ROM SUBROUTINES · QUICK REFERENCE (READ ONLY)

CALL CONVENTION
  On-board subroutines are invoked with RST $38 followed by a subroutine byte.
  Bit 0 of the subroutine byte indicates that register-load parameters follow.
  Example:  FF 51 1E  = RST $38, call SR80, load B=$1E (30 interrupts).

SR14 · INTERRUPT SERVICE
  Executes the timer/sound service every interrupt. Maintains sound, one-second
  input marker, software timers, and database counters.

SR16 · SOFTWARE COUNTERS
  Checks selected counters at $4FD5-$4FDC. Nonzero counters are decremented in
  BCD; expiration bits are reported at $4FDD/$4FDE.

SR18 · SOUND GENERATOR ($050A)
  IX points to a work area, HL to a sound parameter string. Positive records use
  a note length in 1/60-second increments. Writes sound ports $10-$18 and stores
  its continuation state in the Bally database.

SR20 · STOP ALL SOUND ($05FE)
  Clears the sound-generator note length and writes zero to all sound channels.

SR22 · SCREEN PARAMETERS ($03CF)
  Programs the vertical-line count, horizontal split/border register, and screen
  interrupt vector.

SR24 · COLOR SPECIFICATIONS ($01DB)
  Outputs eight chrominance/luminance bytes to port $0B.

SR32/SR34/SR36/SR38
  Graphics preparation and CRT data movement helpers used by the onboard display
  routines.

SR50 · CHARACTER DISPLAY ($07E7)
  Displays one character using horizontal/vertical position and the Appendix A
  character parameters.

SR52 · STRING DISPLAY ($07CA)
  E = horizontal position, D = vertical position, C = display parameters,
  HL = character string. Standard onboard characters use the 5x7 table.

SR54 · BCD NUMBER DISPLAY ($0BF1)
  Displays BCD digits with optional leading-zero suppression and standard or
  custom character definitions.

SR56/SR58/SR60
  CRT address calculation and screen-buffer address helpers.

SR64 · CALCULATOR KEYCODE ($0ACF)
  Converts the raw calculator key number in B to the onboard ASCII keycode in A.

SR66 · PLAYER INPUT ($01AC)
  Reports keypad, one-second marker, four triggers, four joysticks, and four knobs.
  Player 1 trigger is device $14; Player 1 joystick is device $15.

SR68/SR70 · INPUT JUMP TABLES
  Dispatch input devices or input values through a three-byte entry table.

SR72 · WAIT FOR SWITCH ($01BA)
  Blanks the TV, waits for switch input, then re-enables the display.

SR80 · WAIT INTERRUPTS ($001B)
  B = number of interrupts to wait. At 59.94 Hz, B=30 is approximately 0.5 sec.

SR82 · DISPLAY TIME ($0BD2)
  Displays the database minutes/seconds timer using 3x5 characters.

SR84 · UP TIMER ($0C1B)
  Increments a six-digit BCD counter and optionally compares it with the preset
  value at $4FF4-$4FF6.

SR118 · RANDOM NUMBER ($037F)
  A = maximum. Returns a random number below A; A=0 accepts any 8-bit result.

CALL $0200 · TIMER SERVICE
  Called by SR14 every interrupt. Keeps sound time, raises the one-second marker,
  and decrements the minute/second timer at $4FED/$4FEE.

SOUND PORTS
  $10 master divider; $11-$14 tone/vibrato controls; $15 voice A/B volume;
  $16 voice C/noise volume; $17 noise control; $18 combined sound output.

BALLY DATABASE
  $4FCE-$4FD4 sound state
  $4FD5-$4FDC eight BCD software counters
  $4FDD-$4FDE counter-expiration flags
  $4FDF-$4FE2 last knob values
  $4FE3 last calculator input / one-second marker
  $4FE4-$4FE7 joystick and trigger state
  $4FE8-$4FE9 color-specification pointer
  $4FEA sound note length
  $4FED-$4FEE minutes/seconds down timer

The interpreter models the BASIC-visible graphics, input, memory, and sound-port
behavior. It does not execute arbitrary Z80 machine code through CALL() in this
first cut.

Decode a cassette

  1. Upload a WAV or drop it directly on the BASIC editor.
  2. Leave the rate on Auto first; force 300 or 2000 if detection chooses poorly.
  3. Open Tape Signal to inspect the leader, waveform, timing split, bytes, and parser output.

Getting a clean decode

  • Use an uncompressed mono or stereo PCM WAV; the decoder mixes stereo to mono.
  • A complete, steady leader is important. Avoid clipped beginnings and aggressive noise reduction.
  • If Auto fails, try the known tape rate, then Invert signal or disable Schmitt cleanup in Settings.
  • Manual cycle split and start cycle are recovery tools for noisy or nonstandard recordings.

What gets loaded

The stored numbered program becomes editable BASIC source. Tape titles, immediate commands, and loader trailers may appear around it and are ignored by the player.

Patch and modification tapes: load the base program with Replace, choose Merge/patch by line number in Settings, then load the modification tape. Matching line numbers are replaced, new lines are added, and a bare line number deletes that base line.

Run and control

Review or edit the listing, then press Run. Run/Pause/Stop/Reset remain available in the toolbar. BASIC INPUT requests appear directly over the Astrocade display.

Programs that read KP pause at the display for one keyboard character at a time. Type the requested letter, number, or space; no extra Enter is needed. The program resumes the exact statement after each key.

  • Player 1: Arrow keys joystick · Space trigger
  • Player 2: WASD joystick · F trigger
  • Console: Enter GO switch · Esc closes dialogs

Language coverage

Supports integer and character-literal expressions, assignments, IF without THEN, GOTO, GOSUB/RETURN, FOR/TO/STEP/NEXT, PRINT, INPUT, CLEAR, LINE, BOX, RND, ABS, input functions, @() arrays, memory, ports, and core display variables including TV character output and KP keyboard input.

Machine-language compatibility

Known onboard ROM calls are modeled when practical. Unknown CALL routines are skipped with a warning so programs can continue. Programs that depend on custom Z80 code for core graphics, controls, or game logic may still behave incorrectly. Cassette :INPUT stops in a waiting-for-tape state when no chained block is present; :RUN binaries are not executed.
