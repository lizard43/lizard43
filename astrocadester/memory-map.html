Memory Maps

Wizard of Wor·Gorf·Robby Roto·Space Zap

Wizard of Wor

Wizard of Wor has normal low ROM/Magic region, framebuffer RAM at $4000-$7FFF, fixed upper game ROM, a tiny protected-RAM block, and ordinary work RAM for stack and runtime state.

Z80 address space · Wizard of Wor Fixed upper ROM with protected RAM and work RAM near the top.
high addresses at top
Building memory map…
Stack: place SP at $E000 so pushes grow down into high work RAM Protected RAM: only 64 bytes, so reserve it for credits, latches, and tiny bookkeeping. Magic: reads from $0000-$3FFF are ROM reads; writes are graphics operations.

Gorf

Gorf shares the Astrocade arcade foundation, but the upper map is more complex. The $8000-$BFFF window is a dynamic banked region populated by the 91364 ROM/RAM board, while high bookkeeping and workspace move upward into the $E000 range.

Z80 address space · Gorf Banked mission code/data window plus high NVRAM and work RAM.
high addresses at top
Building memory map…
Bank window: the $8000-$BFFF slot is the big difference from Wizard of Wor. NVRAM: High non-volatile/protected storage instead of the WoW-style $D000 block. Port dependency: bank and protected-write behavior must stay tied to the correct I/O model.

Space Zap

Space Zap is the simpler map, it does not use upper ROM expansion or dynamic banking used by Wizard of Wor and Gorf, so large parts of the upper address space remain unmapped.

Z80 address space · Space Zap Small work RAM with upper open-bus regions.
high addresses at top
Building memory map…
Protected RAM: mapped at $D000-$D03F. Stack: place SP near $D800 so pushes grow into $D040-$D7FF. Expansion: the unused $8000-$BFFF area is a useful contrast against WoW and Gorf.

Robby Roto

Keeps the standard low ROM/Magic and framebuffer regions, then uses the upper half of the Z80 address space for fixed game ROM at $8000-$DFFF, high protected/NVRAM storage at $E000-$E7FF, and ordinary work RAM/stack space at $E800-$FFFF

Z80 address space · Robby Roto ROM, NVRAM, and work RAM fill the upper address space
high addresses at top
Building memory map…
Protected RAM: protected window moved up to $E000-$E1FF Stack: ordinary RAM is $E800-$FFFF; place SP at $0000 so pushes grow down into high work RAM NVRAM: Robby Roto keeps NVRAM/protected storage at $E000-$E7FF while $8000-$DFFF is used as 24 KB of upper ROM
